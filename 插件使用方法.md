## 五、常见问题

### ❓ 加载时提示“Manifest 文件无效”
- 检查 `manifest.json` 是否为有效的 JSON 格式（没有多余逗号、引号完整）
- 确认 `manifest_version` 为 `3`

### ❓ 侧边栏不显示
- 确认 Edge 版本 ≥ 114.0
- 检查 `manifest.json` 中是否包含 `"side_panel"` 和 `"permissions": ["sidePanel"]`

### ❓ 图标不显示
- 确认 `icons/` 文件夹内有对应尺寸的图标文件
- 图标文件应为 PNG 格式

### ❓ 修改代码后没有生效
- 在 `edge://extensions/` 页面中，点击扩展卡片上的 **“重新加载”** 按钮（🔄）
- 或关闭并重新打开侧边栏


## 六、升级插件

当你获取到新版源码后：

1. 备份当前规则（使用“设置”面板中的 **导出规则** 功能）
2. 用新文件替换旧文件
3. 在 `edge://extensions/` 页面点击 **“重新加载”**
4. 如需保留规则，导入之前导出的 JSON 文件


## 七、卸载插件

1. 打开 `edge://extensions/`
2. 找到“📚 书籍简介文字处理工具”
3. 点击 **“删除”** 按钮
4. 确认删除


## 八、数据存储说明

插件使用浏览器本地存储（`localStorage`）保存以下数据：

| 存储键名 | 内容 |
|----------|------|
| `bookEditorRules_v3` | 关键词规则列表 |
| `bookEditorRegexRules_v3` | 正则规则列表 |
| `bookEditorDraft_v3` | 输入区草稿内容 |
| `bookEditorTheme_v2` | 主题偏好设置 |

这些数据仅保存在你的本地浏览器中，卸载扩展后数据会被清除（建议提前导出规则备份）。


## 九、注意事项

- 项目配合 **PoxenStudio/mybooks** 项目使用
- 开发者模式下的扩展在每次浏览器启动时会提示“已禁用开发人员模式扩展”，点击“保留”即可继续使用
- 输入内容中的 HTML 标签会被转义处理，避免 XSS 风险
- 建议定期导出规则备份，防止意外丢失
